# Windows 日志与事件查看器

> 一款面向普通用户的 Windows 系统诊断工具，让你无需学习复杂的事件查看器，也能快速了解电脑的运行状态、安全状况和异常活动。

## 为什么需要这个工具？

Windows 自带的事件查看器功能强大，但对普通用户来说：

- 界面复杂，不知道看什么
- 事件 ID 全是数字，看不懂含义
- 没有汇总分析，无法快速判断风险

这个工具就是为了解决这些问题而生 —— **一键诊断，通俗易懂**。

## 功能概览

### 一键异常检测

打开程序即可看到综合安全分析报告，包含：

| 检测项 | 说明 |
|---|---|
| 登录失败 | 是否有人尝试破解你的密码 |
| 深夜登录 | 谁在非工作时间（22:00-06:00）用过电脑 |
| 远程桌面 | 是否有人远程连接过你的电脑 |
| USB 设备 | 历史上插过哪些 U 盘或移动硬盘 |
| 系统崩溃 | 有没有蓝屏、异常关机、突然断电 |
| 网络连接 | 当前哪些程序在连接外部地址 |
| 文件变动 | 指定日期内哪些文件被修改过 |
| 安全建议 | 根据风险等级给出具体的操作建议 |

风险评分 **0-100 分**，一目了然：

- 🔴 **高风险 (≥60)** — 立即改密码，检查不明软件
- 🟡 **中等风险 (30-59)** — 存在值得关注的异常
- 🟢 **低风险 (10-29)** — 少量异常，建议留意
- ✅ **正常** — 未发现异常

### 详细日志查看

提供 **10 个分类标签页**，覆盖系统运行的方方面面：

| 标签页 | 你能看到什么 |
|---|---|
| 系统日志 | 崩溃、服务故障、关机原因、驱动问题 |
| 安全日志 | 登录/注销、权限变更、账户操作 |
| 应用日志 | 程序报错、安装/卸载记录 |
| 登录记录 | 谁登录过、用什么方式、什么时间 |
| USB 历史 | 插过哪些 USB 设备、什么时候插的 |
| 文件改动 | 指定日期内哪些文件被修改/创建/删除 |
| 进程历史 | 运行过什么程序、谁运行的、什么时候 |
| 网络连接 | 当前活跃的 TCP 连接、哪些程序在联网 |
| 已装软件 | 电脑上安装了哪些软件 |
| 系统信息 | 主机名、CPU、内存、磁盘、服务等 |

### 便捷操作

- **日期选择** — 支持最近 30 天任意一天
- **实时搜索** — `Ctrl+F` 快速过滤数据
- **列排序** — 点击表头按任意字段排序
- **查看详情** — 双击任意行查看完整信息
- **导出数据** — 右键导出为 CSV 或 TXT
- **刷新数据** — `Ctrl+R` 一键刷新
- **事件翻译** — 事件 ID 自动翻译为中文含义

## 快速开始

### 方式一：直接运行（推荐）

1. 从 [Releases](https://github.com/你的用户名/Windows-Log-Viewer/releases) 下载最新版 `.exe`
2. **右键 → 以管理员身份运行**
3. 查看「异常检测」标签页，了解电脑安全状况

> **为什么需要管理员权限？** 部分安全日志（如登录记录、权限变更）只有管理员才能读取。非管理员运行仍可正常使用大部分功能。

### 方式二：从源码运行

```bash
# 1. 克隆仓库
git clone https://github.com/你的用户名/Windows-Log-Viewer.git
cd Windows-Log-Viewer

# 2. 直接运行（无需安装任何依赖）
python log_viewer.py
```

> 仅需 Python 3.9+，使用的全是标准库，无需 `pip install`。

### 方式三：自行打包

```bash
pip install pyinstaller
pyinstaller --noconsole --onefile --name "Windows日志事件查看器" log_viewer.py
```

打包后的 `.exe` 文件在 `dist/` 目录下。

## 使用场景

### 场景一：怀疑有人动过我的电脑

1. 打开程序，查看「异常检测」
2. 关注「登录失败」和「深夜登录」
3. 切换到「登录记录」查看具体时间和方式
4. 切换到「文件改动」查看哪些文件被修改过

### 场景二：电脑突然蓝屏/死机

1. 切换到「系统日志」
2. 查看崩溃时间点附近的错误事件
3. 关注来源为 `BugCheck`、`Kernel-Power` 的事件

### 场景三：怀疑中了病毒

1. 查看「异常检测」的风险评分
2. 切换到「进程历史」查看有没有可疑程序
3. 切换到「网络连接」查看有没有异常的外连
4. 切换到「已装软件」查看有没有不明软件

### 场景四：想知道谁插过 U 盘

1. 切换到「USB 历史」
2. 查看所有插过的 USB 设备名称和时间

## 技术架构

```
┌─────────────────────────────────────────┐
│              GUI (Tkinter)              │
├─────────────────────────────────────────┤
│          数据获取层 (DataFetcher)        │
│  ┌─────────┬──────────┬──────────────┐  │
│  │PowerShell│ wevtutil │  注册表/WMI  │  │
│  │Get-WinEvent│        │              │  │
│  └─────────┴──────────┴──────────────┘  │
├─────────────────────────────────────────┤
│          Windows 系统底层               │
│  事件日志 | 注册表 | WMI | 网络状态     │
└─────────────────────────────────────────┘
```

- **GUI 框架**：Tkinter（Python 内置，无需额外安装）
- **数据获取**：PowerShell `Get-WinEvent`、`wevtutil`、注册表查询、WMI
- **打包工具**：PyInstaller（单文件打包，内置所有依赖）

## 常见问题

**Q：为什么部分日志看不到？**
A：请以管理员身份运行程序。安全日志中的登录记录、权限变更等需要管理员权限才能读取。

**Q：加载数据很慢怎么办？**
A：日志数据量大时，首次加载可能需要几秒到几十秒。程序会在加载时显示提示，请耐心等待。低配置电脑可能稍慢。

**Q：可以在 Windows 7 上运行吗？**
A：可以。程序已内置 Windows 7 所需的 CRT 运行库 DLL，无需额外安装。

**Q：如何导出日志？**
A：在任意表格中右键，选择「导出为 CSV」或「导出为 TXT」。
